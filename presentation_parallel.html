Le processeur

  • Est le composant du PC qui effectue les calculs et instructions
  • Il est divis en coeurs (core)
  • un coeur ne peut traiter q'une instruction la fois
  • R utilise par dfaut qu'un coeur
  • Utiliser plusieurs coeurs peut augmenter la vitesse de calcul

circuit integr du i7-2600k

Boucle foreach

Contrairement for retourne des rsultats, stocks par dfaut dans une liste.

Possibilit de changer la faon dont les rsultats sont assembls en passant une fonction .combine :

x <- foreach(a = 1:3, b = rep(10,3), .combine = c) %do% (a + b)
x
## [1] 11 12 13

Remarque : on peut itrer plusieurs variables comme un mapply

Doparallel

Obtenir le nombre de coeurs de la machine

nbcoeur <- detectCores() # intel i5 : 4 coeurs

Il faut tout d'abord lancer une session de R par coeur

clust <- makeCluster(nbcoeur-1) # recommand

Et que ces autres sessions soient detectes par foreach

registerDoParallel(clust)

Renvoie le nombre de coeur qui seront utiliss

getDoParWorkers()
## [1] 3

Stopper le clustering

stopCluster(clust)

%dopar%

Pour parallliser: remplacer %do% par %dopar%

Paralllisation efficace

Efficace si:

  • l'opration rpter est longue (opration droite de %dopar% est longue calculer)
  • le nombre d'itration est un multiple du nombre de coeurs utiliss
foreach(i=(petit ou grand vecteur))%dopar%{Gros modle enerv}
foreach(i=1:3,) %dopar%{glm(NDVI~PP+TEMP,datanasa[[i]],
                                  family=binomial(logit))}
formula1=formula(NDVI~PP)
formula2=formula(NDVI~PP*TEMP)
formula3=formula(NDVI~PP*TEMP + SNOW)
listform=list(formula1,formula2,formula3)
foreach(F=listform) %dopar%{glm(F,datanasa[[i]],
                                  family=binomial(logit))}

Paralllisation

Contre productif si:

  • le temps de calcul est dja court
  • le calcul est long cause d'un grand nombre d'itrations de la boucle, mais qu'une seule itration est trs rapide calculer
foreach(i=1:GROSNOMBRE,.combine = "c"){ Opration ultra rapide}

foreach(i=1:100000,.combine = "c"){ 1+i}

  tfe <- system.time(
    res <- foreach(j = 1:1000, .combine = c) %dopar% sqrt(j)
    )[[3]]
  
  tf <- system.time(
    for(k in 1:1000){
      res2  <- c(res2, sqrt(k))
      }
    )[[3]]
  
  dft[i,] <- c(tfe, tf)
}
